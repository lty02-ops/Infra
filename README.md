## 1세대 컴퓨터 (1940년대 ~ 1950년대)
1세대 컴퓨터는 **진공관(Vacuum Tube)** 을 기반으로 동작했으며 계산, 저장, 제어 용도로 사용(군사용, 과학)했습니다.
진공관은 전기 신호를 증폭하고 스위칭하는 역할을 하지만, 크고 깨지기 쉬우며 열이 많이 발생했습니다.

-   크기가 크고 무거움

-   발열이 심함

## 2세대 컴퓨터 (1950년대 ~ 1960년대)

진공관 대신 **트랜지스터(Transistor)** 를 사용하였고 트랜지스터는 반도체 기반 소자로, 더 작고 효율적이며 신뢰성이 높았습니다.

→ 컴퓨터 성능이 이전에 비해 크게 개선

<br>

### 특징

1세대와 비교해서

-   성능 향상
-   더 작아짐
-   더 빨라짐
-   전력 소모 감소
-   발열 감소
-   신뢰성 향상


<br>
<br>

## 3세대 컴퓨터 (1960년대 ~ 1970년대)

**집적회로(IC, Integrated Circuit)** 를 도입했으며 여러 개의 트랜지스터를 하나의 칩에 집적하였습니다.

→ 성능은 높이고 크기는 더 줄임

특히, 이때 **운영체제의 등장**으로 컴퓨터를 보다 쉽게 사용할 수 있게 되었으며 **다중 프로그래밍**이 가능해졌습니다.

→ 사용자 편의성 향상

<br>

## 특징

-   운영체제 등장
-   다중 프로그래밍 가능
-   속도 크게 향상
-   비용 감소
-   크기 감소
-   신뢰성 증가
-   초기 도입 비용 존재

→ 현대 컴퓨터 구조의 기반 형성

<br>
<br>
<br>

→ 컴퓨터의 소형화 및 대중화

<br>

## 특징

-   개인용 컴퓨터(PC) 등장
-   GUI(그래픽 사용자 인터페이스) 발전
-   인터넷과 함께 급속 성장
-   크기 매우 작음
-   성능 매우 높음
-   가격 저렴
-   대중화 성공

# CPU

## CPU란 무엇인가?

CPU(Central Processing Unit)는 이름 그대로 연산과 제어의 정점에 서 있는 컴퓨터의 최상위 구성 요소입니다.

흔히 사람의 **‘두뇌’** 에 비유되곤 하는데, 이는 CPU가 프로그램의 복잡한 명령어를 **실시간으로 해석하고 실행**하는 핵심 동력을 제공하기 때문입니다.

그 구체적인 역할은 크게 세 가지 흐름으로 요약할 수 있습니다.

-   **데이터의 가공:** 입력 장치를 통해 유입되는 파편화된 정보들을 논리적으로 분석하여 **유의미한 데이터로 전환**합니다.
-   
<br>

-   **프로세스의 조율:** 처리된 결과가 정확한 시점에 출력 장치로 전달될 수 있도록 일련의 과정을 정교하게 통제합니다.

<br>

-   **시스템의 지휘:** 하드웨어와 소프트웨어 사이에서 모든 자원의 우선순위를 결정하는 **'디지털 지휘자'** 역할을 수행합니다.


<br>

우리가 화면에서 보는 모든 움직임과 작업은 CPU라는 지휘자가 1초에 수억 번 이상 내리는 의사결정으로 인한 결과물이라고 볼 수 있습니다.

<br>
<br>

## CPU 구성 요소

### 세 가지 주요 구성 요소

### 1. 제어 장치 (Control Unit, CU) - "지휘자"

**컴퓨터 시스템의 작동을 통제하고 지시하는 장소**

-   **역할:** 메모리(ROM) 에서 명령어를 가져와(Fetch) 해석(Decode)하고, 연산 장치나 메모리 등 다른 부품들에 작업 지시를 내림

<br>

-   **구성** : PC(프로그램 카운터), 명령 해독기, 부호기 ,명령 레지스터 등

<br>
<br>

### 2. 산술논리연산장치 (Arithmetic Logic Unit, ALU) - "계산기"

**명령어를 실행하기 위한 데이터 처리가 일어나는 장소**

- **역할:** 제어 장치의 지시에 따라 덧셈, 뺄셈 같은 **산술 연산**과 AND, OR, NOT 같은 **논리 연산**을 수행함

<br>

- **구성** : ACC(누산기), BREG(버퍼 레지스터) 등

<br>
<br>

### 3. 레지스터 (Register) - "임시 메모장"

**CPU 내부에 있는 소규모의 고속 기억 장치**

- **역할:** 연산에 필요한 데이터나 계산 결과, 명령어의 주소 등을 잠시 동안만 저장하며, CPU 외부의 메모리(RAM)보다 훨씬 빠르지만 용량은 매우 작음

<br>

- 구성 : 범용 레지스터, 특수 목적 레지스터

<br>
<br>

## CPU의 연산 과정

<img src="https://img2.quasarzone.com/editor/2025/11/17/1780f4172b5912a8e24a1de8ca812239.png"  width="400">
<br>

> 출처 : https://quasarzone.com/bbs/qf_cmr/views/2850852

**CPU의 연산 과정은 Fetch → Decode → Execute → Writeback으로 이루어짐**

1.  **인출(Fetch)**
    -   메모리상의 프로그램 카운터(PC)가 가리키는 명령어를 CPU로 인출하여 적재
    
2.  **해석(Decode)**
    -   CU가 명령어를 분석하여, 명령어의 종류와 목적을 파악
    
3.  **실행(Execute)**
    -   ALU가 해석된 명령어에 따라 데이터에 대한 연산을 수행
    
4.  **쓰기(Writeback)**
    -   처리 완료된 데이터를 메모리에 기록
        
        

## CPU의 성능 지표

### **쿨럭 속도**

**CPU가 1초당 처리하는 사이클 횟수**

→ 쿨럭 속도가 높을수록 많은 명령어를 빠르게 처리 가능

<br>
<br>

### **코어 수**

**CPU 내에 있는 물리적인 연산 핵심 장치의 수**

**개수 기준**

-   **싱글코어** : 하나의 CPU 안에 하나의 Core가 내장되어 있는 것
-   **듀얼 코어** : 하나의 CPU 안에 2의 Core가 내장되어 있는 것
-   **멀티 코어** : 여러개의 Core를 가진 CPU

**성능 기준**

**P코어 (Performance)**
- 고성능
- 무거운 작업 담당 (게임, 렌더링)
<br>

**E코어 (Efficiency)**

- 저전력
- 가벼운 작업 담당 (백그라운드, 웹 등)

ex) 직원

<br>

### **스레드 수**

**하나의 코어가 동시에 처리할 수 있는 스레드의 수**

→ 스레드 : 코어가 작업을 처리하는 작업 단위 / 흐름

ex) 일(업무)

<br>

### **캐시 메모리**

<img src="https://velog.velcdn.com/images/ajm0718/post/0036956a-d786-449e-94e9-959c9efa73a4/image.png"  width="400">

> 출처 :https://velog.io/@ajm0718/%EB%A9%94%EB%AA%A8%EB%A6%AC-%EA%B5%AC%EC%A1%B0
<br>

**CPU 내부에 위치한 초고속 메모리**

→ L1(가장 빠름), L2, L3(가장 큼) 단계로 구분 되며, CPU는 이를 통해 데이터를 미리 쟁여

<br>

### **IPC**

**쿨럭 사이클 당 처리할 수 있는 명령어 수**

→ IPC가 높을수록 더 많은 일을 처리함

<br>

### **TDP**

**CPU가 최대 부하 상태에서 발생하는 열을 시키기 위해 필요한 냉각 시스템의 용량**

→ TDP가 높을수록 고성능 쿨러가 필요

<br>

### **메모리 대역폭**

**CPU와 RAM 간 데이터 전송 속도**

→ 높은 대역폭은 데이터 접근 속도를 높여 성능을 향상시킴

<br>

### **벤치마크 점수**

**일련의 테스트를 통해 측정된 성능 점수**

→ 싱글 코어와 멀티 코어 점수를 구분해서 봐야 함

# GPU

## GPU란 무엇인가?

<img src="https://library.gabia.com/wp-content/uploads/2024/09/%EA%B7%B8%EB%A6%BC1-1024x766.png"  width="400">

> 출처 : https://library.gabia.com/contents/infrahosting/13541/

GPU(Graphics Processing Unit)는 컴퓨터 그래픽 및 이미지 처리를 빠르게 하기 위해 설계된 병렬 연산 장치입니다. CPU와 달리 명령어를 순서대로 처리하지 않고 병렬 처리에 최적화되어 있습니다.

초기 목적 : 3D 그래픽, 영상 랜더링 → 현대에는 AI, 데이터 처리, 과학 계산 등에 활용
<br>

## CPU와 GPU 차이점

<img src="https://raw.githubusercontent.com/GRuuuuu/hololy-img-repo/main/2024/2024-02-05-gpu-basics.md/5.png"  width="400">

> 출처 : https://gruuuuu.hololy.org/ai/gpu-basics/

<br>

### CPU

- 소수의 코어지만 성능이 강력하고 명령 전환이 빠름

- 복잡하고 다양한 연산에 유리

→ 범용적

### GPU

- 코어 수 많고 병렬 처리 탁월

- 단순하고 반복적인 연산을 빠르게 수행

→ 병렬 처리용 (고성능)

<br>
<br>

## GPU를 AI에 쓰는 이유

#### 1. 병렬 처리 능력

GPU는 **수천 개의 코어를 동시에 사용**해 반복적인 계산을 병렬로 처리

AI 모델은 수많은 행렬 연산과 뉴럴 네트워크 계산을 필요로 하는데, GPU가 효율적

**ex)** GPT 같은 대규모 언어 모델, 이미지/영상 처리
<br>

#### 2. 성능 대비 효율 우수

CPU 대비 훨씬 빠른 속도 + 높은 에너지 효율

대규모 모델 학습/추론에서 비용과 시간 절감 가능

**ex)** 최신 GPU → AI 추론 성능 1,000배 향상
<br>

#### 3. 대규모 시스템 확장 용이

GPU는 슈퍼컴퓨터 수준으로 확장 가능

NV링크, 인피니밴드 등 고속 인터커넥트로 여러 GPU 연결 → 초대형 AI 모델 처리 가능

<br>

#### 4. AI 모델 성장에 필수

AI 모델(LLM 등) 파라미터가 수십억~수조 단위 → CPU만으로는 불가능

GPU는 수천 코어 + 최적화 메모리 → AI 모델을 단일 GPU 또는 GPU 클러스터에서 처리 가능

→ 초기에는 3D 그래픽, 영상 랜더링용으로 개발, but 현대에는 그래픽이 아닌 분야에서도 사용

ex) 딥러닝, 데이터 분석

### GPU의 연산 처리 방식

GPU 코어 하나 → **1 스레드(Thread) 처리**

수많은 스레드를 묶어 Warp(NVIDIA)/Wavefront(AMD) 단위로 병렬 처리

동작 방식: SIMT(Single Instruction Multiple Threads)

한 명령어를 **여러 스레드가 동시**에 수행

<br>

#### 1. GPU 코어와 스레드

GPU 코어 하나 = **1개의 스레드(Thread)** 를 실행

> 스레드(Thread) = 작은 실행 단위로, CPU에서 말하는 ‘작업 단위’와 비슷

**ex)** 10000개의 데이터에 대해 같은 연산을 해야 한다면

스레드 1개 → 데이터 1개 계산

코어 수 많으면 한 번에 여러 데이터를 동시에 처리 가능
<br>

#### 2. 스레드 묶음: Warp / Wavefront

- 현실적으로 GPU 코어는 수천 개지만, 처리할 스레드는 수십~수백만 개

- 해결 방법: 스레드를 묶어서 한 번에 묶음 단위로 처리

| 제조사 | 용어 | 특징 |  
|:------:|:----:|:----|  
| NVIDIA | Warp | 32개의 스레드 단위로 동시에 실행 |  
| AMD | Wavefront | 64개의 스레드 단위로 동시에 실행 |

- 이 단위로 같은 명령어를 동시에 수행

한 스레드라도 **데이터가 준비되지 않았거나 지연**되면, GPU가 **다른 스레드를 먼저 처리** → 코어 활용 효율 극대화

<br>

### 3. SIMT (Single Instruction, Multiple Threads)

GPU의 핵심 동작 원리
-   **한 명령어를 여러 스레드가 동시에 수행**
-   **ex)**
    -   행렬 곱 연산에서 각 원소 계산 명령어가 동일
    -   Warp 단위로 32개의 스레드가 각각 다른 원소 계산 → 병렬 처리

CPU와 비교:
-   CPU = Single Instruction, Single Thread (SISD)
→ 순차 처리
-   GPU = SIMT → 반복적 연산을 동시에 처리
→ 수천 배 빠름

### 4. 처리 과정

<img width="600" height="400" alt="b7cd7fcb-f883-4e08-a713-fa654b75d083" src="https://github.com/user-attachments/assets/f15945a2-8b6d-4e02-af65-7d8929e1f428" />

1.  데이터 1,000,000개
2.  스레드 1,000,000개 생성
3.  GPU 코어 1,024개
4.  1,000,000 스레드를 **1,024개 코어에 할당 → Warp 단위로 순차적으로 처리**
5.  모든 스레드 연산 완료 → 결과 메모리에 기록

-   즉, **동일한 연산을 다수 데이터에 동시에 적용 → GPU 병렬 처리의 핵심**

# RAM

### RAM은 무엇인가?

RAM(Random Access Memory)은 CPU가 데이터를 임시로 저장하고 빠르게 접근할 수 있도록 돕는 휘성 메모리입니다.

즉, 컴퓨터 전원이 켜져 있는 동안만 데이터가 유지되며, 전원이 꺼지면 데이터가 모두 사라집니다.

<br>

### 하드 드라이브, ROM과 RAM 비교

| 특성 | 하드 드라이브 (HDD/SSD) | ROM | RAM |  
|:----:|:-----------------------:|:---:|:---:|  
| 저장 | 장기 저장 | 영구 저장 | 임시 저장 |  
| 속도 | 느림 | 느림 | 빠름 |  
| 읽기/쓰기 | 가능 | 주로 읽기 | 가능 |  
| 용량 | 매우 큼 | 크지만 제한적 | 상대적으로 적음 |  
| 데이터 손실 | 비휘발성 | 비휘발성 | 휘발성 |

<br>

### RAM 종류

#### 1. SRAM (정적 RAM)

- **캐시 메모리**로 사용

- 트랜지스터 4~6개로 구성, 주기적 refresh 필요 없음
> 주기적 refresh:  커패시터에서 전기가 새기 때문에 데이터가 없어지기 전에 전기를 채워줘야 함

- 밀도(단위 면적당 메모리 셀 수) 가 낮음

- **빠르지만 비싸고** 발열이 많음

<br>

#### 2. DRAM (동적 RAM)

- **메인 메모리**로 사용

- 트랜지스터 + 커패시터 1개로 구성, 주기적 refresh 필요

- 밀도(단위 면적당 메모리 셀 수)가 높음

- **SRAM보다 느리지만 저렴하고** 대용량 저장 가능
  
<br>
<br>

### RAM 특징

| 구분 | 내용 |
|:----:|:----|
| 장점 | **빠른 데이터 접근**<br>**멀티태스킹 효율 향상**<br>**전원 꺼지면 자동 삭제**<br>쉽게 업그레이드 가능 |
| 단점 | **전원 꺼지면 데이터 손실**<br>비싸고 용량 제한 있음<br>지속적인 전력 필요<br>물리적 공간 필요 |

# DISK

## DISK란 무엇인가?

운영체제, 응용 프로그램 및 사용자 데이터와 같은 디지털 정보를 **저장하고 검색하는데 사용**되는 저장 장치입니다.
즉, “컴퓨터의 창고” 역할을 합니다.
<br>
## DISK 종류

<br>

**'플래터'** 라고 불리는 자기 디스크를 **물리적으로 회전**시켜 **데이터를 읽고 쓰는** 전통적인 방식의 저장 장치

-> LP 플레이어가 레코드판 위에서 바늘을 움직여 음악을 재생하는 것과 유사한 원리

<br>

### 구성 요소

**Platter** : 데이터가 저장되는 곳으로 디스크를 의미

<br>

**Spindle** : 플래터를 회전시킴

<br>

**Arm** : Head가 원하는 곳으로 이동시킴

<br>

**Head** : 플래터의 데이터를 읽거나 씀

<br>

### 저장 단위와 관리 방식

#### 플래터의 저장 단위



**Sector**: 플래터를 피자 조각처럼 나누었을 때 생기는 가장 작은 조각. 데이터 저장의 최소 물리 단위(전통적으로 512Byte)

<br>

**Track**: 플래터 중심에서 일정한 거리에 있는 동심원 형태의 길. 육상 트랙처럼 하나의 트랙 위에 수많은 섹터가 나열됨

<br>

**Cylinder** : 여러 장의 플래터가 수직으로 쌓여 있을 때, 수직 상으로 같은 위치에 있는 트랙들의 모임. 헤드가 움직이지 않고 접근할 수 있는 물리적 기준이 됨

<br>

**Cluster**: 연속된 섹터의 모음으로, 운영체제(OS)가 파일을 할당하고 관리하는 가장 작은 논리적 단위

<br>

| 구분 | 내용 |
|:----:|:----|
| 장점 | **낮은 비용**<br>가용성 |
| 단점 | **느린 성능**<br>약한 내구성<br>높은 전력 소비<br>소음 발생<br>진동 발생 |
| 주요 용도 | 단순 데이터 백업<br>영상 보관 |

<br>


### SSD

SSD는 움직이는 부품 없이 반도체(NAND Flash Memory)를 사용하여 데이터를 저장하는 최신 방식

-> 스마트폰이나 USB 메모리와 비슷한 원리

<br>

### 구성 요소

**NAND Flash Memory** : **비휘발성 메모리**로 장기간 보관이 가능

<br>

**Controller** : SSD의 **‘두뇌’** 로, 데이터의 일기, 쓰기 등의 모든 작업을 관리

<br>

**DRAM Cache** : SSD가 데이터를 **임시로 저장하는 메모리 공간**으로, 데이터 접근 속도를 크게 향상시킴

<br>

### 저장 단위와 관리 방식

<img  src="https://webapi3.adata.com/storage/quiktips/img4.jpg"  width="600">
<br>

> 출처 : https://www.adata.com/kr/quikTips/33/


**Cell** : **데이터를 저장**하는 가장 기초적인 물리 소자. 전하를 가두어 0과 1을 기록하며, 적층 방식에 따라 SLC, MLC, TLC 등으로 나뉨

<br>

**Page** : **여러 개의 셀이 모인 단위**로, 데이터를 **읽고(Read) 쓰는(Write) 최소 단위**. OS와 데이터를 주고받는 실질적인 기본 규격

<br>

**Block** : **여러 개의 페이지가 모인 묶음**으로, 데이터를 **지우는(Erase)** 최소 단위. SSD는 덮어쓰기가 안 되기 때문에 반드시 블록 단위로 먼저 지운 후 데이터를 써야함

<br>

**Plane / Die**: **블록들이 모여 이루는 더 큰 물리적 구역**. 다이(Die)는 우리가 흔히 보는 검은색 반도체 칩 형태의 최종적인 물리 단위.

<br>

| 단점 | **높은 비용** |
| 주요 용도 | 시스템 부팅<br>게임<br>작업용 |

메인보드는 컴퓨터 내부에서 가장 큰 **회로 기판(PCB)** 으로, CPU, RAM, 저장장치 등 모든 부품을 연결하고 **데이터와 전력을 전달하는 중심 역할**을 합니다.

→ 컴퓨터의 **모든 부품이 소통**하도록 만들어주는 핵심 허브

<br>

### 주요 특징
-   전기 신호로 **부품 간 데이터 전달**
- **전력 분배** 담당
- 시스템 성능과 확장성 결정

<br>
<br>

**메인보드는 크기와 구조에 따라 다양한 형태로 나뉨**

### ATX (표준형)

- 가장 널리 사용되는 메인보드
- 확장성, 성능, 안정성 균형


<br>

### Micro-ATX (중형)

- ATX보다 작은 크기
- 가성비 중심


<br>

### Mini-ITX (소형)

- 매우 작은 크기 (17 × 17cm)
- 공간 절약
- 저전력, 저소음
- 슬롯 수 적음



<br>
<br>

## 주요 기능

### 1. 데이터 흐름 관리

- CPU, RAM, 저장장치 간 데이터 전달
- 입력 → 처리 → 출력 과정 제어

<br>

### 2. 전력 분배

- 전원 공급 장치(PSU)의 전기를 각 부품에 적절히 분배

<br>

### 3. 부품 간 통신 지원

- 모든 하드웨어가 서로 협력하도록 연결

<br>

### 4. 시스템 성능 향상

- 확장 슬롯을 통해 GPU, RAM 추가 가능
- 기능 확장 및 업그레이드 지원

<br>

### 5. 하드웨어 초기화 및 부팅

- BIOS/UEFI가 시스템 시작 담당

<br>

### 6. 안정성 및 발열 관리

- 전력 제어 + 쿨링 구조로 시스템 보호

<br>

### 7. 확장성과 생산성 제공

- 다양한 장치 연결 가능
- 사용자 작업 효율 향상

→ 벽 콘센트의 **AC(교류)** 전기를 컴퓨터가 사용하는 **DC(직류)** 전기로 변환

<br>

## 주요 기능

- **전압 변환** (AC ↔ DC)
- **보호 기능 수행 (과전압/과전류 등)**
- 전압 안정화 (리플 제거)
- 전류 조절

→ **“안정적인 전력 공급 + 하드웨어 보호”**

1.  **AC 입력**
    -   벽 콘센트에서 110V / 220V 교류 입력

<br>

    -   노이즈 및 전기적 간섭 제거
  
<br>

    -   높은 전압 → 낮은 전압으로 감소

    -   콘덴서로 전압 흔들림 제거

    -   일정한 전압 유지 (부하 변화 대응)

7.  **출력**
    -   CPU, GPU, RAM 등에 안정적 DC 공급

## AC vs DC

### AC (교류)

-   **전압 방향이 계속 바뀜**
-   **장거리 전송에 유리**
-   컴퓨터에는 부적합

<br>

### DC (직류)

-   **전압 일정**
-   CPU, GPU, RAM 필수 전원

<br>

**→ PSU는 “불안정한 AC → 안정적인 DC”로 바꾸는 장치**

<br>
<br>

## 전원 공급 방식

### **전원 공급 방식 2가지**

### 1. 선형 전원 공급 장치

### 동작 방식

-  **변압기 → 정류 → 필터 → 전압 조정**
-  **아날로그 방식으로 전압을 직접 낮춤**

<br>

### 특징

- 출력 전압 **매우 안정적**
- 전기적 노이즈 거의 없음

<br>

### 단점

- **효율 낮음 (30~60%)**
- 발열 큼 (열로 에너지 손실)
- 크고 무거움

<br>

### 사용 분야

- 오디오 장비
- 의료/실험 장비 (노이즈 민감)

→ 컴퓨터에는 거의 사용 안 함

<br>
<br>

### 2. 스위칭 모드 전원 공급 장치 (SMPS)

### 동작 방식

1.  **AC → DC 변환**
2.  고속 스위칭 (ON/OFF 반복)
3.  고주파 변압
4.  필터 및 피드백 제어 (PWM)

<br>

### 특징

- **고효율 (70~95%)**
- 소형 / 경량
- 발열 적음
- 다양한 전압 대응 가능

<br>

### 단점

- 전자기 노이즈(EMI) 발생
- 회로 구조 복잡

<br>

### 사용 분야

- 데스크탑 PC
- 노트북
- 스마트폰

<br>

**→ 현대 컴퓨터 = 거의 전부 SMPS 사용**

# **컴퓨터의 요소들은 어떻게 상호작용이 이루어질까?**


## 1. 입력 (Input)

-   사용자가 마우스를 클릭하거나 키보드를 입력하면 신호가 발생

## 2. 처리 과정 (Processing)

-   입력된 신호는 CPU(중앙처리장치)로 전달
-   CPU는 명령을 해석하고 필요한 연산을 수행함

## 3. 임시 저장소 (Memory - RAM)

-   CPU는 작업 중 필요한 데이터를 RAM에 저장하고, 필요할 때 빠르게 불러와 사용

## 4. 장기 저장 (Storage - SSD/HDD)

-   필요한 경우 데이터는 저장장치에서 RAM으로 불러오거나, 처리 결과를 저장장치에 기록함

## 5. 출력 처리 (Graphics Processing)

-   시각적인 작업이 포함된 경우, CPU는 해당 작업을 GPU에 전달함
-   GPU는 그래픽 데이터를 처리(렌더링)함

## 6. 출력 (Output)

-   처리된 결과는 모니터를 통해 화면에 표시되거나, 스피커를 통해 소리로 출력됨

# 현재 트랜드는 어떻고, 어떤 회사가 유명하고, 서버용 부품으로는 어떻게 만들어졌을까?

## CPU

**AI 협력 구조 :** CPU = GPU를 보조하며 전체 시스템 제어 핵심

**Inference 확대 :** AI 추론·에이전트 처리에서 CPU 역할 증가

**자동 최적화 :** Turbo/Boost로 CPU가 스스로 성능 조절

**안정성 중심 :** 오버클럭보다 효율·발열·수명 중요

**병목 핵심 :** GPU 성능 상승 → CPU·메모리가 병목 요소로 등장

**멀티코어 활용 :** 16코어 중심의 효율형 CPU 선호

→ **AI 시대에도 CPU 수요 증가 + 역할 더 중요해짐**

**게임 + 클라우드 성장 :** VR/AR, 클라우드 게임으로 수요 지속

**내장 GPU 증가 :** 저가·모바일 중심으로 확대

## 서버용 부품 vs 일반용 부품

-   **고성능 중심 :** 많은 사용자 요청과 데이터 처리를 위해 높은 연산 성능 필요
-   **안정성 최우선 :** 오류 발생 시 서비스 전체에 영향 → ECC, 이중화 구조 사용
-   **24시간 연속 운용 :** 서버는 항상 켜져 있어야 하므로 내구성과 발열 관리 중요
-   **확장성 :** CPU, GPU, RAM 등을 추가 장착 가능 → 성능 확장 가능
-   **고전력 구조 :** 여러 부품 동시 사용 → 높은 전력과 효율적인 전원 공급 필요
-   **데이터 중심 설계 :** 저장·처리·전송 속도 모두 중요
